HTML


Hyper Text Markup Language je zvaničan jezik web-a. To nije programski jezik nego opisni jezik. HTML koristi tagove, <opening tag > , </closing tag > i <self-closing tag/ > ili <self-closing tag> u html5.
Za svaki sajt ja mogu da vidim kod kojim je napisan, desni klik => page source ili => inspect.
Kodove pišem u nekom tekst editoru, ja najčešce koristim Sublime Text. Kad hoću da sačuvam neki dokument moram obavezno da mu promenim ekstenziju u dokument.html (.html i .htm kažu da je isto). Najbolje bi bilo da sve fajlove za jednu web stranu čuvam u istom folderu.
Pisanje jednog html fajla počinjem sa <!DOCTYPE html>, a to znaži ovaj dokument će biti html5. Zatim obavezno idu html tagovi, to je root element, između kojih se nalazi head i body. U head-u se nalaze informacije za browser, kao što su tagovi za naslov, meta tagovi, style tag, script tag, linkovi ka css, js, jQuery, bootstrap,...
U body tag-u, tačnije između start i end taga, se nalazi sve ono što će biti prikazano na strani. Za sve što želim da prikažem na stranici postoji tag. Uz pomoć javaScript ja mogu da promenim tekst nekog elementa, document.getElementById("idime").innerHTML("Novi tekst");. Sve je povezano!!!


CSS


Cascading Stylesheet Language, je stajling jezik, kaskadni zato što izvršava poslednju naredbu, tačnije prednost ima najbiliži inline style. Za svaki element u html-u postoji selektor u css-u,
selector {
property:value;
}.
Css može biti eksternal (main.css), embeded (u head-u) i inline (za svaki tag posebno). Najbolje je koristiti eksterni, jer možda je nekad potrebno koristiti isti css za više html fajlova, link u head-u ovako izgleda
<link href="url ili path" rel="stylesheet" type="text/css"> .
Ono što je dobro uraditi na početku je reset css, tako što ću sa sajta da preuzmem kod, ali ovako je jednostavnije *{padding:0; margin:0:}, mada sa tim se prestaje kada se koristi bootstrap. Za css je najbitnije shvatiti pozicioniranje elemenata, ali framework kao što je na primer bootstrap će sve to olakšati.

Media queries, @media, služe za postavljanje css-a za različite veličine ekrana, ali i za to sad postoji pomoć, Responsive Web Design je zadužen za lep prikaz sajta na različitim veličinama ekrana. Css može da se menja uz pomoc JavaScript, document.getElementByTagName("ime").style.fontSize="nova vrednost";, ili na primer source nekog elementa document.getElementById("ime").src="novi url";


Less i Sass


Javila se potreba da Css dobije svoju ekstenziju, svoje varijable i funkcije. Zato dolazi do preprocessor jezika kao što su less i sass. Oni se uz pomoć kompajlera prevode u Css. Sass je popularnije od less-a, a i scss je ok. Kako se linkuje? Main.less selector{property:value;}, sintaksa je ista, ali mora da se konvertuje u main.css. Na zvaničnom sajtu less postoji instalacioni kod, $nmp install -g less, koji treba upisati u command prompt. Da bih to mogla da uradim moram da imam instaliran node.js i njegov comand prompt mi je potreban, koji ću pronaći u search. I onda, ima na stranici less-a, komanda koju treba da ukucam u command prompt $lessc style.less.
Prepros je program koji je za windows najbolji, čim prevučem folder u prepros on će sam naći ako postoji fajl koji treba da se kompajluje. Zvuči komplikovano, ali nije. Less olakšava i ubrzava stilizovanje. Kako se koriste varijable u less-u?
@div_color:blue;
div{background-color:div_color;}, e sad to ovako izgleda komplikovano, ali ja na više mesta mogu umesto boje da koristim varijablu, a kada želim da promenim boju ne moram da tražim u css-u gde sam sve postavila boju, dovoljno je da promenim boju u samoj varijabli i tako će biti promenjena boja gde god sam koristila tu varijablu.
Kroz node comand prompt se krećem isto kao i kroz command line, iste komande važe, cd - promeni direktorijum, dir- vidi šta sve ima na tom direktorijumu, lessc- kompajler za main.less u main.css (c:\\User\Milica\Desktop\less>lessc main.less main.css).
Kako se koriste funkcije u less-u? primer:
@bc:red;
.box{
background-color:@bc;
border: 5px solid darken(@bc, 20%);
}, to znači da ću darken() funkcijom postići da mi boja bude tamnija za 20% od boje koja je zabeležena u varijabli. U less-u mogu da se izmiksaju boje.
Sass je glavni prepros, otvori sass i instaliraj. Napravi fajl i prevuci ga do perepros i pronađi mesto gde želiš da ga kompajluješ. Sass ne koristi vitičaste zagrade{}, nego indent, uvlačenje teksta linija koda. Isto postoji pomoć pri kucanju, o tome u tekst editoru.
Kaže Danilo Kad jednom uđes u tehniku kucanja sass ne vraćaš se više na css. :)
Napravi folder, otvori ga u editoru, ctrl + n da sačuvaš fajl kao index.html, pa tako i za main.sass. Otvori prepros koji koristiš i za less i prevucem main.sass, izaberem da mi ga kompajluje u istom folderu.
Paket za autocomplete u bracketsu se zove Brackets Sass Code Hints.


Bootstrap


Bootstrap je CSS framework (već napisan kod), koristi ga veliki procenat sajtova. Olakšava stilizovanje. Slogan bootstrap-a je Mobile first. Kako se koristi, mogu sa zvaničnog sajta da ga preuzmem ili linkom u head-u ka bootstrap-u bez da ga preuzimam. Sve što je potrebno jeste da koristim imena za klase koje koristi bootstrap i tako će izgledati ono sto je na mojoj strani isto kao što je stilizovao bootstrap. Lako je samo treba da se uvežba. Ja mogu i da prepravim nešto putem mog css-a ako želim, bootstrap mi ostaje u pozadini a za neki element dodam id i style koji ja želim da dodam ili izmenim.
Greed options, znači, da svaki red ima 12 kolona, radi lakseg pozicioniranja elemenata. Objašnjenje:
<div class="row">
<div class="col-lg-4">
</div>, to znači u jednom redu ću zauzeti četiri polja za jednu kolonu za lg (large screen).
ImageHover.css ide uz bootstrap, služi za to kako želim da mi se slika ponaša.
Treba proučiti zvaničan sajt bootstrap-a i videti šta sve tamo postoji.


Java Script


JS je scripting programski jezik, sa HTML-om i CSS-om čini jednu celinu pravljenja i manipulisanja web strnicama. jQueri je ekstenzija za javaScript. Uz pomoc javaScript-a web stranice postaju dinamičnije i interaktivnije. JavaScript kod se čuva na serveru ali server šalje celu stranicu browseru (html, css, js), zato se zove Client Side Language. EcmaScript je u stvari JavaScript, Ecma se isto ponaša u svim browserima. Java nije isto što i JavaScript.
Gde implementirati javaScript? Java Script je običan tekst, to znači da mu ne treba kompajler. Kompajleri su programi koji prevode kod. Kod će biti smešten u html fajlu, između otvorenog i zatvorenog script taga, ali je bolje da imam poseban fajl, main.js, i link ka fajlu u script tagu.

<script type="text/javascript" src="main.js"></script>


jQuery


jQuery je najbolji način za pisanje javaScript, sa zvanične strane uzimam kod i uključiću ga u moj projekat, mogu i da preuzmem jQuery i smestim na moj kompjuter. jQuery je javaScript biblioteka. Kompatibilan je sa CSS3, u svakom brauzeru će isto da se ponaša, ne moram da razmišljam da li će nešto da izgleda drugačije na drugom browseru. Na zvaničnom sajtu, mogu da preuzmem fajl (download uncompresed) i da ga linkujem u head-u
<script src="jquery.js"></script>.
Queri znači selektovati nešto, kako doći do DOM (Document Object Model) elementa. Sintaksa jQuery-a je:
jQueri('h1').css('color','red');
i promeniće boju mog h1.
Drugi način da uključim jQueri u moj projekat je da linkujem sa Hosted Library, tu se nalaze sve popularne jS biblioteke, kopiram kod i ubacim ga u source skript taga. U istom fajlu mogu da pišem i jS i jQuery kod. Dovoljno je da koristim dolar znak $, jer je za funkciju, function jQuery(){}, postavljena varijabla, var $ = jQuery. Na sajtu jQueryIPA postoje sve funkcije koje mogu da se koriste.


EcmaScript


Ecma Script v6 je skup svih najboljih funkcija iz svih programskih jezika na jednom mestu. U EcmaScript se umesto apostrofa i navaodnika koriste back-ticks, dve tačkice na dugmetu ispod esc na tastaturi. Ranije sam nešto pisala ("Na stari nacin:" + imevarijable + "."), a u ecma ('novi nacin sa back-ticks:' $(imevarijable).);, rezultat će biti isti. Između dva apostrofa 'use strict' na početku koda, me uslovljava da koristim pravila pisanja, to je zahtev od browsera. Skraćenica let je isto sto i var, varijabla, samo nije vidljiva za global scope, nego za function scope.


XML


Extensible Markup Language, je proširena verzija html-a. Kreiran je za čuvanje i prenos podataka, i što je najvažnije, koriste ga svi programski jezici. Sintaksa je slična kao i kod html-a, imam jedan root tag, ostali su ugneždeni. Svi tagovi, atributi i vrednosti imaju proizvoljna imena, tj. ona koja im sama dodelim.
<?xml version="1.0" encoding="UTF-8"?> je obavestenje za browser da ocekuje main.xml fajl.
AJAX XML DTD je gramatika,
<!DOCTYPE note[<element note(to, from, heading, body)>
<!element to(#pcdata)>
<!element from(#pcdata)>
<!element heading(#pcdata)>
<!element body(#pcdata)>]>, to znači da će moj dokument imati note element za root, i unutar njega elemente to, from, heading, body, koji će biti nekog tipa podataka (pcdata). To je primenjeno u XHTML, služi za notifikacije kada se prijavljujem na neki sajt, XML aplikacija je RSS kratka verzija neke vesti koja vodi na link te vesti.


AJAX


Asynchronous JavaScript and XML je komunikacija sa serverom bez reload-a stranice, to znači ako kliknem na like ili napišem komentar automatski postaje vidljiv na strani, to omogućava XMLHttp objekt. Novi način pristupa, služi se da uz pomoć jS uzme podatke od XML putem servera. AJAX dobro komunicira i sa JSON formatom. jQuery.ajax() je metod (funkcija). U html fajlu će se nalaziti link do jQuery.js i main.js. U main.js fajlu moram da upišem jQueri(document).ready(function){ fajl.xml, njegov link kopiram ovde};.
Za ready() funkciju ubacim $.ajax(), za URL ubacim adresu koju sam kopirala, type="get" i dataType="xml".


JSON


Java Script Object Notation i XML su osnovne tehnologije kada se razmenjuju podaci, share. JSON je inspirisan jS objektima i koriste ga svi IT jezici. JSON objekt mora da bude unutar vitičastih zagrada {} i navodnika. Brojevi, boolean i null ne moraju u navodnike. Može da bude string, array[ ], numbers, boolean, null, ali ne moze da sadrži funkciju. JSON služi da dobijem podatke putem web-a. Ima svoju zvaničnu stranicu.

Text editor


Uglavnom su svi tekst editori dobri, ja koristim Brackets a najčesce Sublime text. Brackets je baš fokusiran na web dizajn, u ExtensionMng mogu naći razne dopune koje olakšavaju kucanje koda, nalazi se sa desne strane. Postoji i live preview, veza sa google chrom-om. Na ctrl + e dobijam instant edit, tu mogu direktno da stilizujem i pratim uživo promene, esc za izlaz.
Sublime je super, ima multi kursor ctrl + klik na svako mesto gde želim isti tekst da pišem. Paketi u sublime koje ja koristim su : ctrl + shift + p (ovim otvaram settings) => instal pac control, restartujem sublime.
Zatim ctrl + shift + p => instal paket emmet => ok, kad imam instaliran emmet mogu da kucam .ime + tab i automatski ću dobiti div klase ime, ili li*5 + tab dobiću pet list item.

Sledeće ću instalirati jQueri plug-in, ctrl + shift + p => instal (nadji paket jQuery), tako da ću sada u main.js moci da kucam dolar znak $ + tab i otvara mi ceo kontekst.
Sledeće, ctrl + shift + p => instal advance new file, ovo omogućava da ne moram new file => new folder nego ctrl + alt + n. U tekst editoru dole na liniji ću imati mogućnost da kucam css\main.css i to ce mi kreirati i folder i fajl za Css.

Git i GitHub


Git je softver, source version kontrol (omogućava kretanje kroz faze) i collaboration (saradnja više programera). Instaliranje Git-a, preuzmem ga sa zvanične stranice , desni klik => git Bash here, to je novi oblik terminala. Sada u njemu treba da proverim da li sam instalirala git tako sto ću ukucati
git--version, na isti način mogu i u command prompt ili power shell da proverim.

Na GitHub moram da se registrujem, tu onda mogu da smestim moj kod. BitBucket je sličan kao GitHub. GitHub je web sajt baziran na osnovama softvera Git. Tu mogu da gledam tuđe fajlove, da ih preuzmem, da radim ja nešto na njima, da ih vratim vlasniku sa promenama. Source version control omogućava kretanje kroz grane (branches). Kad postavim neki folder on se nalazi na master branch, kad izvršim neku promenu onda to smestam na druge grane. Moram da uradim sinhronizaciju gitHub, sa lokalnog da prebaci na globalni.

Npm install angular bootstrap angular- route, (node.js), komanda, je najlakši način da instaliram frameworks, da ne moram da ih skidam i linkujem ručno. Jedino što će onda biti dosta fajlova koje nije potrebno da neko, ko želi da preuzme moj kod, preuzima. To se rešava time što se napravi folder .gitignore, i opet sinhronizacija.

Konačno, strana se postavlja na GitHub Pages, to je besplatni host, server.
